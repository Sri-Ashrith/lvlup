VERCEL PROJECT ENV VARIABLES (Frontend + Serverless API)

JWT_SECRET=replace_with_a_long_random_secret
ADMIN_PASSWORD=replace_with_admin_password
CLIENT_ORIGIN=https://your-vercel-domain.vercel.app
SUPABASE_URL=https://your-project-ref.supabase.co
SUPABASE_SERVICE_ROLE_KEY=replace_with_service_role_key


OPTIONAL CLIENT ENV VARIABLES (set only if needed)

VITE_API_URL=
VITE_ENABLE_SOCKET=false
VITE_SOCKET_URL=


NOTES

1) Keep VITE_API_URL empty to use same-origin /api on Vercel.
2) Set VITE_ENABLE_SOCKET=false on Vercel (recommended for serverless runtime).
3) Do NOT expose SUPABASE_SERVICE_ROLE_KEY in client code.
4) This setup uses only Vercel (hosting + API) and Supabase (database).
